# Web Request Trace — 数据溯源

Chrome 扩展（Manifest V3），专为后台管理系统前端日常排查设计。

点击页面任意元素，反查它显示的数据来自后端哪个接口的响应体——告别"这个字段是哪个接口返回的？"的灵魂拷问。

## 特性

### 数据溯源

点击页面上的数据（如某个用户名、订单号、状态标签），扩展自动：

- 检索已捕获的全部接口响应体
- 将点击元素的文本内容与响应数据进行匹配
- 按匹配度排序，展示最可能的接口来源
- 高亮响应体中命中的文本片段

同时支持 `fetch` 和 `XMLHttpRequest`，零配置开箱即用。

### 接口探测

表单提交前，开启**探测模式**再点提交按钮。扩展会**拦截并阻断**即将发出的请求，展示：

- 请求方法和 URL
- 请求头
- 请求体（表单数据、JSON 等）

请求不会真正发到后端——适合"看看这个按钮到底提交了什么参数"的场景，不会有副作用。

### 全局开关

点击扩展图标，一键开启/关闭数据溯源功能（所有标签页生效），开启时图标显示 **ON** 角标。

## 安装

1. 下载或克隆本仓库
2. 打开 Chrome，访问 `chrome://extensions/`
3. 开启右上角的**开发者模式**
4. 点击**加载已解压的扩展程序**，选择项目目录
5. 工具栏出现 Web Request Trace 图标

## 使用指南

### 数据溯源

1. 点击扩展图标开启功能（图标显示 **ON** 角标）
2. 页面右下角出现 **🔎 数据溯源** 浮动按钮
3. 点击按钮进入选择模式
4. 点击页面上任意数据元素——弹出浮层展示匹配的接口列表
5. 点击接口路径复制 `METHOD /path?query` 到剪贴板
6. 展开条目查看命中的响应片段
7. 按 **Esc** 退出选择模式

### 接口探测

1. 开启扩展（图标显示 **ON** 角标）
2. 点击 **🎯 接口探测** 浮动按钮
3. 提示："接口探测已就绪——去点按钮，将拦截并显示它要请求的接口"
4. 点击任意按钮或提交表单
5. 浮层展示被拦截的请求详情
6. 请求**已被阻断**，刷新页面即可恢复正常

### 手动搜索

溯源面板的搜索框中可手动输入任意文本，回车查找哪个接口的响应包含该文本。

## 项目结构

```
├── background.js      # Service Worker：全局开关、状态持久化、跨标签广播
├── content-script.js  # 隔离桥接：注入 inject.js，传递开关状态
├── inject.js          # 核心运行时：hook fetch/XHR、匹配算法、Shadow DOM UI
├── manifest.json      # MV3 清单文件
└── icons/             # 扩展图标
```

## 工作原理

数据溯源匹配算法：

1. 从选中元素的文本中提取有效 token（中文词组、字母数字值）
2. 滤除噪音（常见 UI 文案：编辑/删除/详情、通用词：data/list/name）
3. 对每个已捕获的接口响应，计算包含多少个命中的 token
4. 按匹配分数降序排列返回结果

## 隐私说明

所有 API 响应数据仅保存在浏览器标签页的**内存**中，不会发送到任何服务器。`storage` 权限仅用于记录开关状态。
